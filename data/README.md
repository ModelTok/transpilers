# C++ → Mojo dataset (behaviorally verified)

A small, **faithful** C++→Mojo translation dataset mined from EnergyPlus and
verified by execution. Built by `scripts/build_cpp_mojo_dataset.py`.

## Why this exists — and why it is *not* mined from energyplus-mojo

The obvious source of C++→Mojo pairs would be the energyplus-mojo repo, which has
both EnergyPlus C++ and hand-written Mojo kernels. **That pairing is poisoned.**
The Mojo kernels are spec-driven reimplementations, not faithful translations:

> `ashrae_simple_h_in.mojo` keeps **3 of the 7** convection coefficients in the
> real C++ `CalcASHRAESimpleIntConvCoeff` (drops the tilted-surface 3.870/2.281
> branches) and inverts a sign convention. The faithful 7-coefficient form is in
> neither the Mojo nor the Python layer of that repo.

Training on such pairs teaches *wrong* translations. So this dataset is built by
**generate-and-verify** instead of mining:

```
real EnergyPlus C++ scalar fn
  → transpilers (algorithmic C++→Mojo, no LLM for pure-arithmetic fns)
  → compile BOTH standalone, run on ~125 threshold-aware sampled inputs
  → (a) outputs agree to rel-err ≤ 1e-9 on ≥4 finite points  AND
    (b) every *computational* branch of the C++ was exercised (gcov gate)
  → keep only pairs that pass both
```

Fidelity is guaranteed by the behavioral gate **plus a branch-coverage gate**.
The coverage gate is load-bearing: without it, a transpiler bug in an
unexercised branch would ship under a "verified" label. Concretely,
`CalcASHRAESimpleIntConvCoeff` has 7 convection coefficients across 5 branches;
naive sampling (cosines drawn from 0.5–300, but a cosine is bounded to [−1,1])
"verified" it at rel-err 0 *without ever executing the 3.870 tilted-enhanced
branch* — the very branch distinguishing the faithful C++ from the poisoned
energyplus-mojo kernel. Threshold-aware sampling (probe around each numeric
literal in the C++ body) + the gcov gate now exercise every computational
branch.

Each pair records its coverage mode:
- `"full"` — every executable line of the C++ body was hit.
- `"computation-full"` — every line *with computation* was hit; the only
  uncovered lines are zero-measure constant-return fallbacks guarded by exact
  float equality (e.g. `return 3.076;` reached only when `DeltaTempCosTilt==0.0`
  exactly). These carry negligible transpile risk (the transpiler renders every
  constant return identically) and the uncovered count is reported, not hidden.

The Mojo is idiomatic scalar Mojo (`def f(x: Float64) -> Float64`), the correct
transpiler-output form — not an FFI batch kernel.

## Scope

Self-contained pure-scalar functions: `Real64 Name(Real64 const a, ...)` with
only scalar const args (Real64/Nandle/Real32/double/float/int/Int/Int64/long/
bool), no `EnergyPlusData` / arrays / references / pointers / strings. The
extractor is a paren+brace-matching scanner that handles multi-line and
inline-commented signatures (pervasive in EnergyPlus), and the standalone-compile
gate drops anything that secretly depends on another translation unit.

This is exactly the transpiler's verified-capable domain. It does **not** cover
stateful subroutines, array kernels, or class methods — those need a different
(verified) pipeline.

Current yield: **40 verified pairs** from 107 extracted candidates across 16
EnergyPlus source files (28 transpile failures, 39 rejected by the
behavioral/coverage gate). Regenerate with the commands below; counts will track
the EnergyPlus + transpiler versions.

## Files

| File | Format | Use |
|---|---|---|
| `cpp_mojo_pairs.jsonl` | one JSON object per pair (see schema) | analysis, eval, custom training |
| `cpp_mojo_sft.json` | alpaca `{instruction, input, output}` | LLaMA-Factory SFT (`cpp_mojo_to_sft.py`) |

### Pair schema (`cpp_mojo_pairs.jsonl`)

```json
{
  "cpp_source": "...full C++ function text incl. signature+braces...",
  "mojo_source": "def CalcSatVapPressFromTemp(Temp: Float64) -> Float64: ...",
  "function_name": "CalcSatVapPressFromTemp",
  "source_file": "ThermalComfort.cc",
  "n_args": 1,
  "arg_types": ["Real64"],
  "ret_type": "Real64",
  "verification": {"method": "behavioral", "samples_total": 125,
                   "samples_finite": 122, "max_rel_err": 8.0e-11,
                   "cpp_lines_covered": 9, "cpp_lines_uncovered": 0,
                   "branch_coverage": "full"},
  "provenance": "energyplus-cpp-generate-verify",
  "direction": "cpp->mojo"
}
```

## Provenance / licensing

- C++ source: EnergyPlus (BSD-3-Clause, U.S. DOE / LBNL et al.). Function bodies
  are derived works of that source — retain EnergyPlus attribution if you
  redistribute.
- Mojo side is machine-generated by this repo's transpiler.
- Build reads EnergyPlus + the energyplus-mojo Mojo toolchain **read-only**; no
  source repo is modified.

## Reproduce

```bash
uv run python scripts/build_cpp_mojo_dataset.py --out data/cpp_mojo_pairs.jsonl
uv run python scripts/cpp_mojo_to_sft.py
```
